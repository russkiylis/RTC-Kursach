# Курсовая по РТЦиС в MATLAB

Программа автоматизирует расчёты для курсовой работы по дисциплине
«Радиотехнические цепи и сигналы»: строит заданный видеоимпульс, считает
спектры, синтезирует согласованный фильтр, рассчитывает шумы, отношения
сигнал/шум и квазиоптимальный RC-фильтр.

## Самый простой запуск, если вы совсем впервые

### 1. Установите MATLAB

Нужен обычный MATLAB. Дополнительные тулбоксы для базового запуска не
требуются.

### 2. Скачайте проект с GitHub

Есть два способа.

**Способ 1: через Git**

Откройте Terminal на macOS или PowerShell/Git Bash на Windows и выполните:

```bash
cd Desktop
git clone https://github.com/russkiylis/RTC-Kursach.git
cd RTC-Kursach
```

После этого на рабочем столе появится папка `RTC-Kursach`.

**Способ 2: без Git, через ZIP**

1. Откройте страницу проекта: <https://github.com/russkiylis/RTC-Kursach>.
2. Нажмите зелёную кнопку `Code`.
3. Нажмите `Download ZIP`.
4. Распакуйте архив.
5. Полученную папку можно переименовать в `RTC-Kursach`.

### 3. Откройте папку в MATLAB

1. Запустите MATLAB.
2. В панели `Current Folder` откройте папку `RTC-Kursach`.
3. Важно: текущей папкой MATLAB должна быть именно корневая папка проекта,
   где лежат файлы `Koshcheev_var6.m`, `Tihonov_var6.m` и папка
   `@KursachSolver`.

### 4. Запустите нужный вариант

Откройте нужный файл варианта и нажмите `Run` или клавишу `F5`.

Например, для варианта Кощеева:

```matlab
Koshcheev_var6
```

Или просто откройте `Koshcheev_var6.m` и нажмите `Run`.

После запуска MATLAB начнёт открывать окна с графиками и писать численные
результаты в Command Window.

## Какой файл запускать

Запускаются только файлы вариантов в корне проекта:

- `Koshcheev_var6.m` — вариант 6, Кощеев;
- `Tihonov_var6.m` — вариант 6, Тихонов;
- `Logunova_var7.m` — вариант 7, Логунова;
- `Hvorova_var18.m` — вариант 18, Хворова;
- `Hozyainov_var19.m` — вариант 19, Хозяинов.

Файлы внутри папки `@KursachSolver` напрямую запускать не нужно. Это
внутренний код расчётов.

## Как сделать свой вариант

1. Скопируйте любой готовый файл варианта, например `Koshcheev_var6.m`.
2. Переименуйте копию, например в `Ivanov_var12.m`.
3. В начале файла измените параметры:

```matlab
U1 = 6;
U2 = 6;
U3 = 4;
U4 = 0;
T = [1 3 4].*1e-6;
n = 14;
m = 40;
f_gr = "max";
T2 = 5e-6;
```

Что означают параметры:

- `U1`, `U2`, `U3`, `U4` — уровни кусочно-линейного видеоимпульса;
- `T` — три возможных значения \(T_1\), из которых выбирается сигнал;
- `T2` — полная длительность сигнала;
- `n` — параметр уровня шума;
- `m` — параметр ширины полосы шума;
- `f_gr` — критерий выбора сигнала: `"min"`, `"mid"` или `"max"`.

Остальной код обычно менять не нужно.

### Как менять точки tau для RC-фильтра

В каждом файле варианта точки для таблиц и графиков RC-фильтра заданы явно:

```matlab
rcTauFactors = 0.1:0.1:2.4;
rcTau0 = 1 / abs(solver.f_gr01_FFT(solver.selectedSignal));
rcTauValues = rcTauFactors .* rcTau0;

solver.p41_calcRCResponses(rcTauValues);
```

Здесь:

- `rcTau0` — опорная постоянная времени \(\tau_0 = 1/f_{0.1}\);
- `rcTauFactors` — множители относительно \(\tau_0\);
- `rcTauValues` — реальные значения \(\tau\) в секундах.

Например, строка

```matlab
rcTauFactors = 0.1:0.1:2.4;
```

означает, что программа посчитает точки
\(0.1\tau_0, 0.2\tau_0, ..., 2.4\tau_0\).

Если нужны другие точки, меняйте только `rcTauFactors`. Например:

```matlab
rcTauFactors = [0.1 0.2 0.5 1.0 1.5 2.0 2.4];
```

или так:

```matlab
rcTauFactors = 0.1:0.05:2.5;
```

Внутренние файлы `@KursachSolver` для этого менять не нужно.

## Что программа считает

### Дано

- строит три возможных входных сигнала;
- строит СПМ входного шума;
- выводит параметры шума.

Методы:

```matlab
solver.dano_showSignal();
solver.dano_showNoiseSPM();
```

### Пункт 1

- считает спектры через БПФ;
- строит производные сигнала;
- строит аналитический спектр;
- выбирает нужный сигнал по критерию `f_gr`.

Методы:

```matlab
solver.p1_showSpectrFFT();
solver.p1_showSignalDiff();
solver.p1_showSpectrAnalytical();
```

### Пункт 2

- синтезирует согласованный фильтр;
- строит неупрощённую и упрощённую структурные схемы;
- строит АЧХ и ФЧХ согласованного фильтра;
- строит временные диаграммы по узлам схемы.

Методы:

```matlab
solver.p2_createSF();
solver.p2_simplifySF();
solver.p2_showSF();
```

### Пункт 3

- строит корреляционную функцию выбранного сигнала;
- считает выходной сигнал согласованного фильтра;
- считает СПМ шума на выходе СФ;
- считает дисперсию и СКО шума на входе и выходе;
- считает отношения сигнал/шум;
- рассматривает прямоугольный импульс той же энергии;
- печатает LaTeX-формулы КФ и выполняет проверки.

Методы:

```matlab
solver.p33_showCorrFunc();
solver.p32_showOutputNoiseSPM();
solver.p33_showNoiseStats();
solver.p34_showSNR();
solver.p35_showRectResponse();
solver.p36_showRectSNR();
solver.p33_showCorrFormulas();
solver.p33_verifyCorrFunc();
```

### Пункт 4

- считает выходной сигнал RC-фильтра для набора постоянных времени;
- считает дисперсию и СКО шума на выходе RC-фильтра;
- строит зависимость отношения сигнал/шум от \(\tau\);
- уточняет настоящее оптимальное значение \(\tau_{RC,opt}\);
- сравнивает оптимальный RC-фильтр с согласованным фильтром;
- строит выход RC-фильтра при прямоугольном видеоимпульсе;
- строит СПМ шума на выходе RC-фильтра при \(\tau_{RC,opt}\).

Методы:

```matlab
rcTauFactors = 0.1:0.1:2.4;
rcTau0 = 1 / abs(solver.f_gr01_FFT(solver.selectedSignal));
rcTauValues = rcTauFactors .* rcTau0;

solver.p41_calcRCResponses(rcTauValues);
solver.p44_calcRCNoiseStats("stats");
solver.p45_calcRCSNR();
solver.p46_showRCOptimalComparison();
solver.p49_showRectRCResponse();
solver.p44_calcRCNoiseStats("spm");
```

## Как отключить лишние графики

Программа открывает много окон. Если нужен только один раздел, лишние
строки можно временно закомментировать знаком `%`.

Например, чтобы не строить пункт 4:

```matlab
% rcTauFactors = 0.1:0.1:2.4;
% rcTau0 = 1 / abs(solver.f_gr01_FFT(solver.selectedSignal));
% rcTauValues = rcTauFactors .* rcTau0;
% solver.p41_calcRCResponses(rcTauValues);
% solver.p44_calcRCNoiseStats("stats");
% solver.p45_calcRCSNR();
% solver.p46_showRCOptimalComparison();
% solver.p49_showRectRCResponse();
% solver.p44_calcRCNoiseStats("spm");
```

Чтобы вернуть расчёт, уберите `%`.

## Где брать числа для отчёта

Большая часть численных результатов печатается в MATLAB Command Window.
Там появляются:

- выбранное значение \(T_1\);
- граничные частоты;
- формулы спектра и передаточных функций в LaTeX;
- дисперсии и СКО шума;
- отношения сигнал/шум;
- оптимальное значение \(\tau_{RC,opt}\);
- максимальные значения выходных сигналов.

Графики можно сохранить через окно Figure:

```text
File -> Save As
```

Для Word обычно удобно сохранять в PNG.

## Частые проблемы

### MATLAB пишет, что не найден `KursachSolver`

Скорее всего, MATLAB открыт не в той папке. В `Current Folder` должна быть
папка `RTC-Kursach`, внутри которой видна папка `@KursachSolver`.

### Открылся миллион графиков

Это нормально: программа строит почти весь курсовик. Закройте ненужные окна
или закомментируйте лишние строки в файле варианта.

### Нужно поменять вариант

Меняйте только верхний блок параметров в файле варианта. Внутренние файлы
`@KursachSolver` трогать не нужно.

### График выглядит странно после изменения параметров

Проверьте:

- все времена заданы в секундах, например `1e-6`, а не `1`;
- `T2` больше любого значения из `T`;
- `f_gr` записан строкой: `"min"`, `"mid"` или `"max"`;
- после изменения файла нажмите `Run` заново.

## Краткая логика расчёта

1. По заданию строится кусочно-линейный видеоимпульс.
2. По уровню 0.1 от максимума спектра выбирается нужный сигнал.
3. Для выбранного сигнала аналитически считается спектр.
4. По спектру синтезируется согласованный фильтр.
5. Через корреляционную функцию находится выход СФ.
6. Рассчитывается шум на входе и выходе СФ.
7. Для сравнения рассчитывается прямоугольный импульс той же энергии.
8. Для RC-фильтра подбирается оптимальная постоянная времени.
9. Сравниваются СФ и квазиоптимальный RC-фильтр.
